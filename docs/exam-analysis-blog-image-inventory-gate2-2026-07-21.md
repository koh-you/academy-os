## 원본 수집 계약

- 본문 범위는 `se-main-container`를 기준으로 잘라 댓글·프로필·푸터 이미지를 제외한다.
- 스티커, 이모티콘, 지도, 설문, 프로필, 로고 썸네일은 제외한다.
- 네이버 본문 `src`는 보통 `type=w800`이므로 같은 공개 자산의 `type=w966` 변형을 요청해 해시와 크기를 계산한다.
- URL의 경로와 슬라이드 순서는 `PostView` 현재 본문을 원본으로 본다.
- manifest에는 URL과 메타데이터만 저장하며 이미지 base64나 바이너리를 넣지 않는다.
- 게시물이 바뀌면 같은 스크립트를 다시 실행하고 SHA-256 차이를 확인한다.

- 공통 CTA 중복 1장: 기존 대표 해시를 재사용
- 1차 코퍼스 내부 중복 5개 첨부: 동일 해시 대표 한 장만 시각 분석하고 모든 출처에 결과 연결
- 최종 시각 분석 단위: 215개 고유 해시

## Gate 3 입력 계약

Gate 3는 원본 사실과 시각 관찰을 분리한다.

1. `imageId`, `logNo`, `slideOrder`, `sha256`, 크기는 Gate 2 manifest에서 읽는다.
2. 기존 Vision 12건은 저장된 결과를 우선 재사용한다.
3. 신규·파생 이미지는 역할, 레이아웃, 컴포넌트, 텍스트 밀도, 원본 문제/풀이 여부를 분석한다.
4. 동일 SHA-256은 한 번만 분석하고 여러 게시물 출처를 연결한다.
5. `observed`와 `inferred`를 별도 필드로 둔다.
6. 문항 번호·점수·교재명·등급컷은 이미지에 보이거나 교사가 확정한 경우에만 사실로 사용한다.
7. 원본 이미지나 블로그 문장을 저장소에 복제하지 않는다.
