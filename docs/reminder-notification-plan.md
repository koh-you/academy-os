# 운영 리마인더 알림 계획

## 결론

MVP에서는 학생 개별 리마인더의 기본 채널을 카카오 알림톡으로 두고, 강사용 운영 리마인더는 매일 오전 9시 슬랙 요약으로 둔다.

웹앱 푸시는 추후 PWA 설치와 알림 권한 동의 흐름이 안정화된 뒤 추가한다. 인스타그램 DM 자동 알림은 학원 운영 알림의 기본 채널로 사용하지 않는다.

## 원본 데이터

운영 알림의 원본은 Supabase `academy_reminders` 테이블이다.

대시보드/수업 허브는 이 원본을 입력·관리하는 중심 화면이다. 학생프로필, 수업일지, 학사일정은 이 원본 row를 조건에 맞게 읽어서 표시한다. 슬랙과 GPT는 원본이 아니라 `academy_reminders`를 읽는 출력 채널이다.

운영 SQL 적용 파일:

```text
supabase/20260710_academy_reminders.sql
```

## 발송 대상

- 학부모: 수업 당일 오후 데일리리포트 알림톡 안에서 재시험/보충 일정 안내
- 학생: 당일 오전, 일정 1시간 전 재시험/보충 리마인더 알림톡
- 강사: 당일 오전 9시 슬랙으로 상담/신입생/특이사항 알림, 재시험, 보충 일정 요약

## 학생 리마인더

백엔드 엔드포인트:

```text
POST /api/notifications/student-schedule-reminder
```

사용 템플릿:

```text
SOLAPI_STUDENT_COMMENT_TEMPLATE_ID
```

변수:

```text
#{학원명}
#{학생명}
#{수업명}
#{수업일}
#{코멘트}
```

예시 코멘트:

```text
[중요] 오늘 재시험 일정이 있습니다.
내용: 공통수학1 누적테스트
일시: 2026-06-17 18:00
수업: 월수금 7-10반
```

## 강사 슬랙 오전 요약

상세 설정 절차는 `docs/slack-integration-guide.md`를 따른다.

백엔드 엔드포인트:

```text
POST /api/notifications/slack-today-schedule
```

환경변수:

```text
SLACK_WEBHOOK_URL
SLACK_DRY_RUN
```

예시 메시지:

```text
[으뜸수학 고태영T] 2026-07-10 오늘 운영 일정

상담/운영 알림
- 학부모 상담 · 김보현 · 기말 이후 상담 · 2026-07-10 14:00 · 1학기 기말 결과 공유

재시험
- 김룡기 · 공통수학1 누적테스트 · 18:00

보충
- 최선호 · 숙제보충 · 17:00
```

## 스케줄 실행 방식

권장 순서:

1. 백엔드 API를 먼저 안정화한다.
2. 상담/신입생/특이사항 알림은 `academy_reminders`, 보충/재시험 일정은 수업/보충 원본에 저장한다.
3. Render Cron 또는 GitHub Actions가 매일 09:00 KST에 백엔드를 호출한다.
4. 오전 요약은 강사 슬랙으로 보낸다.
5. 학생 알림톡은 당일 오전/1시간 전 대상자를 계산해 발송한다.

스케줄 기준:

```text
KST 09:00 = UTC 00:00
cron: 0 0 * * *
```

중복 방지:

`POST /api/notifications/slack-today-schedule`은 실발송 후 `notification_jobs`에 `slack_daily_summary_YYYY-MM-DD` 기록을 남긴다. 같은 날짜에 다시 호출되면 기본값으로 이미 발송된 슬랙 요약을 건너뛴다. 운영자가 강제로 다시 보내야 할 때만 `force: true`를 사용한다.

## 웹앱 푸시 알림

가능하지만 MVP 기본 채널로 두지 않는다.

필요 조건:

- PWA 설치
- 브라우저 알림 권한 허용
- 서비스워커
- Push subscription 저장
- 서버에서 Web Push 발송

주의:

- 학생이 알림 권한을 거부하면 발송할 수 없다.
- 기기/브라우저별 동작 차이가 있다.
- 카카오 알림톡보다 초기 설정 실패 가능성이 높다.

## 인스타그램 알림

학원 운영 리마인더의 기본 발송 채널로 사용하지 않는다.

이유:

- DM 자동 발송 정책 제약이 크다.
- 학생별 일정 리마인더처럼 반복적이고 사적인 운영 알림에 부적합하다.
- 수신 보장성과 감사 로그 관리가 카카오 알림톡보다 약하다.
