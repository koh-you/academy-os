# 알림톡 설계 초안

목표: koh_you_math 웹앱에서 출결, 데일리리포트, 학생별 코멘트를 Solapi 알림톡으로 발송한다.

중요 원칙:
- 실제 발송 전에는 항상 dry-run으로 payload를 확인한다.
- 테스트 단계에서는 모든 수신자를 원장 번호 `01057882748`로 돌린다.
- 출결 알림톡과 데일리리포트 알림톡은 본문 변수를 하나로 두고, 코드에서 조건부 문장을 조립한다.
- 재시험, 보충, 미완료 숙제, 강사코멘트처럼 해당 정보가 없으면 해당 줄은 만들지 않는다.

## 1. 출결 알림톡

### 발송 대상

학부모

### 발송 시점

- 학생이 태블릿 출결 화면에서 등원 처리될 때
- 학생이 태블릿 출결 화면에서 하원 처리될 때
- 강사가 지각/결석/인정결석/사유를 확정했을 때

### 발송하지 않는 경우

- 출결 상태가 `대기`인 경우
- 같은 학생이 짧은 시간 안에 중복 입력한 경우
- 강사가 아직 지각/결석 사유를 확정하지 않은 경우
- `ALIMTALK_DRY_RUN=true` 상태에서는 실제 발송하지 않고 payload만 확인

### 변수

출결도 변수를 많이 두지 않고 `#{출결본문}` 하나로 조립한다.

```text
#{학원명}
#{학생명}
#{출결본문}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 출결 안내

#{학생명} 학생 출결 안내입니다.

#{출결본문}
```

### 출결본문 예시: 등원

등원은 지각분/사유를 표시하지 않는다.

```text
2026-06-17 18:54에 등원 처리되었습니다.
수업: 월수금 7-10반
```

### 출결본문 예시: 하원

```text
2026-06-17 22:01에 하원 처리되었습니다.
수업: 월수금 7-10반
```

### 출결본문 예시: 지각

지각일 때만 지각분과 사유를 표시한다.

```text
2026-06-17 19:18에 지각 처리되었습니다.
수업: 월수금 7-10반
지각: 18분
사유: 학교 수행평가 후 이동
```

### 출결본문 예시: 결석

```text
결석 처리되었습니다.
수업: 월수금 7-10반
사유: 독감
```

## 2. 데일리리포트 알림톡

### 발송 대상

학부모

### 발송 시점

1. 강사가 수업일지에 강사코멘트를 작성한다.
2. AI가 강사코멘트를 학부모용 문장으로 다듬는다.
3. 강사가 최종 확인한다.
4. 강사가 `학부모 발송 예약`을 누른다.
5. 예약된 리포트는 정해진 시간에 일괄 발송된다.

### 일괄 발송 시각

- 평일: 오후 10:30
- 토요일: 오후 6:00

### 발송 전 누락 확인

발송해야 하는 리포트 중 핵심 데이터가 비어 있으면 발송시각 15분 전에 강사에게 알림을 보낸다.

- 평일: 오후 10:15
- 토요일: 오후 5:45

초기 MVP에서는 웹앱 내부 알림으로 시작하고, 추후 Slack 알림으로 확장한다.

### 핵심 데이터

발송 전 확인해야 할 기본 데이터:
- 학생
- 수업일
- 출결
- 지난 숙제 또는 다음 숙제
- AI 수정 후 강사가 확인한 리포트본문

### 발송하지 않는 경우

- 강사가 `학부모 발송 예약`을 누르지 않은 경우
- 핵심 데이터가 비어 있고 강사가 강제 발송하지 않은 경우
- 같은 수업, 같은 학생에게 이미 발송된 리포트를 재발송 확인 없이 다시 누른 경우

### 변수

```text
#{학원명}
#{학생명}
#{수업일}
#{리포트본문}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 데일리 리포트

#{학생명} 학생의 #{수업일} 수업 리포트입니다.

#{리포트본문}
```

### 리포트본문 구성

아래 항목은 데이터가 있을 때만 들어간다.

```text
출결: 출석
지난 숙제: GRIP ~987
다음 숙제: 쎈 643~647
미완료 숙제:
- RPM 629~668
- GRIP 928~957
재시험 일정: 6월 19일 금요일 공수1 누적테스트
보충 일정: 6월 20일 토요일 13:00
코멘트: 경우의 수 조건 해석을 보완하면 좋겠습니다.
```

### 조건부 조립 규칙

- `출결`은 가능하면 항상 포함한다.
- `지난 숙제`는 값이 있을 때만 포함한다.
- `다음 숙제`는 값이 있을 때만 포함한다.
- `미완료 숙제`는 누적된 미완료 숙제가 있을 때 모두 표시한다.
- `재시험 일정`은 재시험이 잡힌 학생에게만 포함한다.
- `보충 일정`은 보충이 잡힌 학생에게만 포함한다.
- `코멘트`는 특이사항이 있거나 강사가 남길 말이 있을 때만 포함한다.

### 강조 표시 한계

카카오 알림톡 일반 템플릿에서는 HTML, 볼드, 형광펜 같은 서식을 자유롭게 넣을 수 없다.

대신 중요한 일정은 문장 앞에 표시어를 붙여 눈에 띄게 한다.

```text
[중요] 재시험 일정: 6월 19일 금요일 공수1 누적테스트
[중요] 보충 일정: 6월 20일 토요일 13:00
```

강조가 꼭 필요하면 웹앱의 학생/학부모 화면에서 카드, 색상, 배지로 강조하고 알림톡은 링크/요약 역할을 맡긴다.

## 3. 학생별 코멘트 알림톡

### 발송 대상

학생

### 발송 시점

- 강사가 학생에게 개별 안내를 작성한다.
- AI가 학생용 문장으로 다듬는다.
- 강사가 최종 확인한 뒤 `학생 발송`을 누른다.

### 발송하지 않는 경우

- 코멘트가 비어 있는 경우
- 학생 전화번호가 없는 경우
- 강사가 학생에게 공개하지 않기로 한 내부 코멘트인 경우

### 분기

- 숙제 안내
- 시험 안내
- 보충 안내
- 재시험 안내
- 개별 피드백
- 일반 공지

### 변수

```text
#{학원명}
#{학생명}
#{수업일}
#{수업명}
#{코멘트}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 학생 안내

#{학생명} 학생에게 보내는 안내입니다.

#{코멘트}
```

## 4. 미완료 숙제 운영 원칙

미완료 숙제는 단일 값이 아니라 학생별 누적 목록으로 관리한다.

필요한 화면:
- 강사 수업일지
- 숙제현황
- 보충관리
- 학생 화면
- 학부모 화면
- 데일리리포트 알림톡

운영 판단:
- 미완료가 계속 누적되면 강사가 보충을 잡는다.
- 숙제양이 과한 경우 다음 숙제량을 조절한다.
- 이미 검사해서 해결된 미완료는 누적 목록에서 완료 처리한다.
- 미검사 상태로 남은 미완료는 다음 수업에도 계속 표시한다.

## 5. 운영 단계

1. 이 문서를 기준으로 문구를 확정한다.
2. Solapi에서 템플릿 3개를 만든다.
3. 템플릿 승인을 받는다.
4. Render 환경변수에 템플릿 ID를 입력한다.
5. `ALIMTALK_DRY_RUN=true`로 payload를 확인한다.
6. `ALIMTALK_ALLOW_REAL_PARENT_NUMBERS=false` 상태에서 원장 번호로 실제 발송 테스트를 한다.
7. 최종 확인 후 실제 학부모/학생 번호 발송을 허용한다.

## 6. Render 환경변수

```env
SOLAPI_API_KEY=
SOLAPI_API_SECRET=
SOLAPI_FROM=
SOLAPI_PFID=
SOLAPI_ATTENDANCE_TEMPLATE_ID=
SOLAPI_DAILY_REPORT_TEMPLATE_ID=
SOLAPI_STUDENT_COMMENT_TEMPLATE_ID=
SOLAPI_DISABLE_SMS=true
ALIMTALK_DRY_RUN=true
ALIMTALK_TEST_RECIPIENT=01057882748
ALIMTALK_ALLOW_REAL_PARENT_NUMBERS=false
```
