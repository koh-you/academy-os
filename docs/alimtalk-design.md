# 알림톡 설계 초안

목표: koh_you_math 웹앱에서 출결, 데일리리포트, 학생별 코멘트를 Solapi 알림톡으로 발송한다.

중요 원칙:
- 실제 발송 전에는 항상 dry-run으로 payload를 확인한다.
- 테스트 단계에서는 모든 수신자를 원장 번호 `01057882748`로 돌린다.
- 데일리리포트는 변수 수를 줄이고 `#{리포트본문}` 안에서 조건부 문장을 조립한다.
- 재시험, 보충, 미완료 숙제처럼 특이 일정이 없으면 해당 문장 줄은 만들지 않는다.

## 1. 출결 알림톡

### 발송 대상

학부모

### 발송 시점

- 학생이 태블릿 출결 화면에서 등원 처리될 때
- 학생이 태블릿 출결 화면에서 하원 처리될 때
- 강사가 지각/결석/사유를 수동 확정했을 때

### 발송하지 않는 경우

- 출결 상태가 `대기`인 경우
- 강사가 출결을 임시 저장만 한 경우
- 같은 학생이 짧은 시간 안에 중복 입력한 경우
- `ALIMTALK_DRY_RUN=true` 상태에서는 실제 발송하지 않고 payload만 확인

### 분기

- 등원
- 하원
- 지각
- 결석
- 인정결석

### 변수

```text
#{학원명}
#{학생명}
#{상태}
#{시간}
#{수업명}
#{지각분}
#{사유}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 출결 안내

#{학생명} 학생이 #{시간}에 #{상태} 처리되었습니다.

수업: #{수업명}
지각/결석 사유: #{사유}
```

### 변수 예시

```json
{
  "학원명": "koh_you_math",
  "학생명": "김룡기",
  "상태": "등원",
  "시간": "2026-06-17 18:54",
  "수업명": "월수금 7-10반",
  "지각분": "0",
  "사유": "해당 없음"
}
```

## 2. 데일리리포트 알림톡

### 발송 대상

학부모

### 발송 시점

- 수업일지에서 학생별 리포트 내용을 확인한 뒤 강사가 `학부모 발송`을 누를 때
- AI가 강사코멘트를 다듬은 뒤 강사가 최종 확인했을 때

### 발송하지 않는 경우

- 강사코멘트가 비어 있는 경우
- 다음 숙제/지난 숙제/출결 등 핵심 데이터가 아직 저장되지 않은 경우
- 강사가 발송 보류로 표시한 경우
- 같은 수업, 같은 학생에게 이미 발송된 리포트를 재발송 확인 없이 다시 누른 경우

### 추천 변수

변수를 많이 만들지 않고 본문을 하나로 합친다.

```text
#{학원명}
#{학생명}
#{수업일}
#{리포트본문}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 데일리 리포트

#{학생명} 학생의 #{수업일} 수업 리포트입니다.

#{리포트본문}
```

### 리포트본문 기본 구성

아래 항목은 데이터가 있을 때만 들어간다.

```text
출결: 출석
지난 숙제: GRIP ~987
다음 숙제: 쎈 643~647
미완료: RPM 629~668
재시험 일정: 6월 19일 금요일 공수1 누적테스트
보충 일정: 6월 20일 토요일 13:00
코멘트: 경우의 수 조건 해석을 보완하면 좋겠습니다.
```

### 조건부 조립 규칙

- `출결`은 가능하면 항상 포함한다.
- `지난 숙제`는 수업일지에 값이 있을 때만 포함한다.
- `다음 숙제`는 수업일지에 값이 있을 때만 포함한다.
- `미완료`는 못한 숙제가 있을 때만 포함한다.
- `재시험 일정`은 재시험이 잡힌 학생에게만 포함한다.
- `보충 일정`은 보충이 잡힌 학생에게만 포함한다.
- `코멘트`는 강사가 작성하거나 AI 수정본을 확정했을 때 포함한다.

### 변수 예시: 일반 수업

```json
{
  "학원명": "koh_you_math",
  "학생명": "김룡기",
  "수업일": "2026-06-17",
  "리포트본문": "출결: 출석\n지난 숙제: GRIP ~987\n다음 숙제: 쎈 643~647\n코멘트: 경우의 수 조건 해석을 보완하면 좋겠습니다."
}
```

### 변수 예시: 재시험 일정이 있는 경우

```json
{
  "학원명": "koh_you_math",
  "학생명": "서정원",
  "수업일": "2026-06-17",
  "리포트본문": "출결: 출석\n지난 숙제: 행렬 프린트 30문제\n다음 숙제: 경우의 수 프린트\n재시험 일정: 6월 19일 금요일 공수1 누적테스트\n코멘트: 계산은 안정적이나 조건 해석을 다시 확인해야 합니다."
}
```

## 3. 학생별 코멘트 알림톡

### 발송 대상

학생

### 발송 시점

- 강사가 학생에게 개별 안내를 작성하고 `학생 발송`을 누를 때
- AI가 학생용 문장으로 다듬은 뒤 강사가 최종 확인했을 때

### 발송하지 않는 경우

- 코멘트가 비어 있는 경우
- 학생 전화번호가 없는 경우
- 강사가 학생에게 공개하지 않기로 한 내부 코멘트인 경우

### 분기

- 숙제 안내
- 시험 안내
- 보충 안내
- 재시험 안내
- 개별 피드백
- 일반 공지

### 변수

```text
#{학원명}
#{학생명}
#{수업일}
#{수업명}
#{코멘트}
```

### 솔라피 템플릿 초안

```text
[#{학원명}] 학생 안내

#{학생명} 학생에게 보내는 안내입니다.

#{코멘트}
```

### 변수 예시

```json
{
  "학원명": "koh_you_math",
  "학생명": "최선호",
  "수업일": "2026-06-17",
  "수업명": "월수금 7-10반",
  "코멘트": "오늘 오답 중 경우의 수 조건 분류 문제를 다시 확인하세요. 다음 수업 전까지 표시한 5문항을 다시 풀어오면 됩니다."
}
```

## 4. 운영 단계

1. 이 문서를 기준으로 문구를 확정한다.
2. Solapi에서 템플릿 3개를 만든다.
3. 템플릿 승인을 받는다.
4. Render 환경변수에 템플릿 ID를 입력한다.
5. `ALIMTALK_DRY_RUN=true`로 payload를 확인한다.
6. `ALIMTALK_ALLOW_REAL_PARENT_NUMBERS=false` 상태에서 원장 번호로 실제 발송 테스트를 한다.
7. 최종 확인 후 실제 학부모/학생 번호 발송을 허용한다.

## 5. Render 환경변수

```env
SOLAPI_API_KEY=
SOLAPI_API_SECRET=
SOLAPI_FROM=
SOLAPI_PFID=
SOLAPI_ATTENDANCE_TEMPLATE_ID=
SOLAPI_DAILY_REPORT_TEMPLATE_ID=
SOLAPI_STUDENT_COMMENT_TEMPLATE_ID=
SOLAPI_DISABLE_SMS=true
ALIMTALK_DRY_RUN=true
ALIMTALK_TEST_RECIPIENT=01057882748
ALIMTALK_ALLOW_REAL_PARENT_NUMBERS=false
```
