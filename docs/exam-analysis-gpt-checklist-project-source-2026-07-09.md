# 시험분석 체크리스트 GPT 프로젝트 소스 - 2026-07-09

이 파일은 ChatGPT 프로젝트의 `소스`에 올리는 교체용 문서다.

프로젝트 `지침`에는 역할, 금지 규칙, 브랜드 톤 같은 고정 규칙을 넣고, 이 파일은 시험별 체크리스트를 어떻게 읽고 카드뉴스 기획/이미지 생성으로 바꿀지 알려주는 기준 소스로 사용한다.

## 목표

Academy OS 시험분석 화면에서 만든 `GPT 대화세션 체크리스트`와 `GPT 기획 패킷`을 바탕으로 블로그/인스타 카드뉴스를 반복 제작한다.

핵심 원칙은 다음과 같다.

- 웹앱과 시험지분석에서 확인 가능한 값은 자동 입력값 또는 후보값으로 사용한다.
- 선생님이 입력/수정/저장한 체크리스트 값이 최종 원본이다.
- GPT는 시험 사실을 새로 추측하지 않고, 선생님 확정값을 보기 좋은 카드뉴스 기획안과 이미지 프롬프트로 바꾼다.
- 주요문항은 GPT가 고르지 않는다. 선생님이 최종 선택한 문항만 카드뉴스에 넣는다.

## 데이터 원천 우선순위

시험분석 카드뉴스에서 사용할 사실은 아래 순서로 읽는다.

1. `선생님 확정 입력`: 웹앱 체크리스트에서 선생님이 직접 입력하고 저장한 값
2. `시험지분석 확정/후보`: 객관식/서술형/단답형 수, 단원별 비중, 난도 흐름, 주요문항 후보
3. `웹앱 자동 입력`: 학교/학년, 고사명, 과목, 시험범위, 기존 총평/메모
4. 빈칸: 확인되지 않은 값은 만들지 말고 `검수 필요`로 남긴다.

금지 원칙:

- 등급컷, 문항번호, 교재명, 페이지, 유사유형 근거를 추측하지 않는다.
- 시험지분석 후보를 선생님 확정값처럼 말하지 않는다.
- 선생님이 수정한 표현을 AI가 임의로 덮어쓰지 않는다.

## 웹앱 체크리스트 구조

Academy OS의 시험분석 산출물 패널에는 아래 구조로 체크리스트를 입력한다.

### A. 웹앱 자동 입력

웹앱에 이미 있는 시험 정보다. GPT는 이 값을 기본 배경으로 읽는다.

- 학교/학년
- 고사명
- 시험일
- 과목
- 시험범위
- 교과서/출판사
- 기존 시험 후 총평
- 수업/보충/상담 연결 메모

### B. 시험지분석 자동 후보/확정

시험지분석 v2에서 읽는 구조 정보다. 선생님 검수 전에는 후보로 취급한다.

- 객관식 수
- 서술형 수
- 단답형 수
- 총점/배점
- 단원별 실제 비중
- 문항별 단원/유형 분포
- 초반/중반/후반 난도 흐름 후보
- 학생들이 흔들릴 지점 후보
- 주요문항 후보

### C. 선생님 확정 입력

GPT가 카드뉴스 원본으로 가장 중요하게 읽어야 하는 값이다.

- 등급컷/예상 등급컷
- 교과서 출제 근거
- 학교 학습지/프린트 출제 근거
- 부교재 출제 근거
- 모의고사/학력평가 출제 근거
- 시중 심화교재 유사유형 근거
- 확실한 근거
- 주요문항 최종 선택 1
- 주요문항 최종 선택 2
- 주요문항 최종 선택 3

주요문항 최종 선택은 아래 항목을 함께 읽는다.

- 문항번호
- 선생님 선택 이유
- 핵심 개념
- 유사유형 근거
- 자주 틀리는 지점
- 추천 복습

### D. 선생님 해석/문장화

시험을 보기 좋게 설명하기 위한 해석 값이다. 너무 분석적으로 딱딱하지 않게, 학부모가 이해하기 쉬운 카드뉴스 문장으로 바꾼다.

- 한 줄 총평
- 시간관리 포인트
- 체감 난도
- 조건 해석
- 계산량
- 글밥/문장제
- 그래프/도형 해석
- 경우 분류
- 서술형 완성도
- 실수 포인트

### E. 다음 대비

시험분석을 학습 방향으로 연결하는 값이다.

- 훈련 1
- 훈련 2
- 훈련 3
- 사용할 교재/부교재
- 수업/상담 연결 포인트

### F. 카드뉴스 톤

카드뉴스와 블로그 문장에 반영할 톤이다.

- 강조 키워드 3개
- 학부모에게 전달할 메시지
- 학원 관리 문장

### G. 금지/불확실

GPT가 반드시 지켜야 하는 제한이다.

- 쓰면 안 되는 교재명
- 확인 안 된 문항번호
- 확인 안 된 등급컷
- AI가 추측하면 안 되는 내용

## 웹앱 사용 위치

Academy OS에서는 아래 순서로 체크리스트를 작성한다.

1. `시험분석` 화면으로 이동한다.
2. 사용할 분석 카드를 선택한다.
3. `블로그/인스타 산출물 초안` 영역을 연다.
4. `GPT 대화세션 체크리스트` 섹션에서 자동 입력값과 시험지분석 후보를 확인한다.
5. 선생님 확정 입력칸을 채운다.
6. 주요문항 최종 선택 블록에서 문항번호, 선택 이유, 유사유형 근거를 입력한다.
7. `산출물 저장`을 눌러 웹앱에 저장한다.
8. `GPT 기획 패킷 복사`를 눌러 ChatGPT 프로젝트 대화세션 첫 메시지에 붙여넣는다.

저장 원칙:

- 체크리스트 입력값은 Academy OS의 시험분석 산출물 저장값이 원본이다.
- 새로고침 후에도 선생님 입력값이 유지되어야 한다.
- GPT 대화세션에서 수정한 내용이 최종본이 되려면 웹앱에도 다시 반영하거나, 선생님이 별도로 최종 확인해야 한다.

## GPT 대화세션 워크플로우

### 1단계. 기획 세션

첫 메시지에는 웹앱의 `GPT 기획 패킷`을 붙여넣고 아래처럼 요청한다.

```text
아래 Academy OS 시험분석 체크리스트를 바탕으로 10장 카드뉴스 기획안을 만들어줘.

아직 이미지는 만들지 말고, 카드별 역할/메인 문구/보조 문구/강조 키워드/검수 필요 사실만 표로 정리해줘.

중요:
- 제공된 체크리스트 밖의 시험 사실은 추측하지 마.
- 주요문항은 선생님 최종 선택 문항만 사용해.
- 불확실한 값은 카드에 넣지 말고 검수 필요로 남겨.
- 벤치마킹 블로그의 구조는 참고하되 문장/브랜드/디자인은 복제하지 마.
```

기획안 출력 형식:

```text
| 카드 | 역할 | 메인 문구 | 보조 문구 | 강조 키워드 | 넣을 숫자/표 | 검수 필요 |
| --- | --- | --- | --- | --- | --- | --- |
```

### 2단계. 카드별 이미지 생성 세션

기획안을 선생님이 수정/확정한 뒤 카드 1장씩 생성한다.

```text
아래 확정 기획안을 바탕으로 카드뉴스 {카드 번호}번 이미지를 만들어줘.

[고정 스타일]
- 으뜸수학 고태영T
- 1080x1350 세로 카드뉴스
- 블루/화이트 중심의 전문적이고 차분한 내신 분석 스타일
- 모바일에서 읽히는 큰 제목과 짧은 문장
- 숫자, 문항 수, 교재명, 키워드는 배지/표/강조 블록으로 정리
- 원본 블로그 디자인을 그대로 복제하지 않고 구조만 참고

[이번 카드]
- 카드 번호:
- 카드 역할:
- 메인 문구:
- 보조 문구:
- 강조 키워드:
- 반드시 넣을 숫자/표:
- 제외할 내용:

[금지]
- 제공되지 않은 시험 사실 추가 금지
- 확인 안 된 등급컷/문항번호/교재명 추가 금지
- 작은 글씨를 빽빽하게 넣기 금지
- 원본 블로그 문장/브랜드/레이아웃 복제 금지
```

### 3단계. 카드 수정 세션

생성 이미지가 마음에 들지 않으면 전체를 다시 설명하지 말고 수정 포인트만 준다.

```text
이 카드에서 아래만 수정해줘.

- 유지할 것:
- 바꿀 것:
- 빼야 할 것:
- 더 강조할 것:
- 텍스트 오류:

시험 사실은 새로 추가하지 말고, 기존 확정 기획안 안에서만 수정해줘.
```

## 10장 카드 기본 역할

카드 수는 기본 10장이다. 학교별로 내용은 달라져도 역할은 크게 흔들지 않는다.

1. 표지: 학교/학년/고사명과 한 줄 총평
2. 시험 구조: 문항 수, 배점, 객관식/서술형/단답형 구조
3. 출제 원천: 교과서, 프린트, 부교재, 모의고사, 유사유형 근거
4. 단원 비중: 많이 나온 단원과 예상보다 두드러진 단원
5. 난도 흐름: 초반/중반/후반 체감 난도
6. 흔들린 지점: 조건 해석, 계산량, 글밥, 그래프/도형, 경우 분류
7. 주요문항 1: 선생님 최종 선택 문항
8. 주요문항 2 또는 주요문항 묶음: 선생님 최종 선택 문항
9. 다음 대비: 훈련 방향과 사용할 교재/부교재
10. CTA: 으뜸수학 고태영T의 관리 방식, 상담/블로그 유입 문장

## 벤치마킹 키워드 변환 축

평범한 시험을 과장하지 않고, 실제 시험 특징을 한눈에 보이게 분류한다.

- 출제 원천: 교과서 중심, 학교 프린트 반영, 부교재 연계, 모의고사 연계, 심화교재 유사유형
- 구조: 객관식 중심, 서술형 없음, 단답형 포함, 후반부 고난도 집중, 배점 편차 큼
- 난도: 초반 안정형, 중반 개념 응용, 후반 조건 해석, 시간 압박, 계산량 증가
- 학생 난점: 조건 해석, 긴 문장제, 그래프/도형 해석, 경우 분류, 부호/계산 실수, 서술형 표현
- 다음 대비: 유형 반복, 유사유형 비교, 조건 정리 훈련, 시간 제한 풀이, 오답 원인 분류

## 통과 기준

GPT가 만든 기획안이나 이미지 프롬프트는 아래 기준을 통과해야 한다.

- 선생님 확정값과 시험지분석 후보를 구분해서 사용했다.
- 주요문항을 임의로 추가하지 않았다.
- 등급컷/문항번호/교재명/페이지를 추측하지 않았다.
- 한 카드에 하나의 역할만 담았다.
- 학부모가 보기에 시험 구조, 특징, 다음 대비가 한눈에 보인다.
- 으뜸수학 고태영T 브랜드로 재구성되어 있고 벤치마킹 블로그를 복제하지 않았다.

## 중단 조건

아래 상황이면 이미지 생성을 진행하지 말고 선생님 확인을 요청한다.

- 주요문항 최종 선택이 비어 있다.
- 유사유형 근거가 불확실한데 카드에 넣으려 한다.
- 등급컷이 확인되지 않았는데 숫자로 넣으려 한다.
- 시험범위/교재명/문항 수가 웹앱 값과 대화세션 값에서 다르다.
- GPT가 시험 사실을 새로 만들어냈다.
- 카드에 텍스트가 너무 많아 모바일 가독성이 떨어진다.
