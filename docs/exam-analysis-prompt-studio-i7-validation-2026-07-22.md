# 시험분석 프롬프트 제작실 I7 검증 및 사람 Gate

## AI E2E 결과

비운영 상계고 고1 2026년 1학기 기말고사 fixture로 아래 전체 흐름을 실행했다.

1. `final_fields`, `teacher_fields`, `ai_fields`, PDF 원천을 I2 snapshot으로 분리
2. 확정값만 프롬프트 local draft의 시작값으로 seed
3. 학교 스타일·배점 구조·문제 crop·검증 손풀이 파일명을 교사 입력으로 보완
4. 고등학교용 문구 사례 1개를 명시 적용
5. expected revision 0으로 저장 payload 생성
6. revision 1의 Supabase 재조회 응답을 비운영으로 모사하고 전체 값 대조
7. 고등학교 시퀀스 12장을 1~12번으로 조립
8. 마스터, 슬라이드별 상세 12개, 수정 12개, QA 12개, 전체 수정 프롬프트 생성

결과는 전체 생성 준비 완료이며 운영 DB write 0회, 유료 AI 호출 0회다. PDF `pageImageManifest`는 문제 crop으로 사용되지 않았다.

## 자동검증 명령

- `npm run test:exam-prompt-studio`
- `npm run build`
- `git diff --check`

신규 시험분석 suite는 모두 통과한다. 전체 `npm run test:production`은 신규 suite와 supplement fixture를 먼저 통과한 뒤, 기존 Windows CRLF 문자열 비교인 `04d-1 student schedule override...` 1건만 실패한다. 시험분석 코드나 동작 실패가 아니며 병렬 유지보수/리팩터링 소유 파일을 이 브랜치에서 수정하지 않는다.

## 사람 검수 필요 판정

필요하다. AI 자동검증만으로 다음 세 가지를 볼 수 없다.

1. 실제 시험분석 화면에서 긴 입력 카드와 프롬프트 영역의 밀도·모바일 가독성
2. 실제 브라우저 클립보드의 한글/줄바꿈 보존 및 TXT/JSON 다운로드
3. 실제 Supabase 환경에서 저장→재조회→새로고침 유지와 revision 충돌 메시지

실제 GPT Image 생성 품질은 이 Gate의 구현 범위 밖이다. 이미지 생성은 유료 호출이므로 자동 실행하지 않는다.

## 사람 검수 순서

1. 리팩터링 브랜치 통합 후 시험분석 작업 하나를 연다. `최종 미리보기` 아래 `GPT Image 슬라이드 프롬프트 제작실`이 보이는지 확인한다.
2. 운영 원본을 바꾸지 않아도 되는 테스트 run을 사용한다. 실제 학생·알림·수업 원천은 건드리지 않는다.
3. 학교 스타일과 비어 있는 배점 구조를 입력하고, 주요문항 1개에 실제로 GPT 대화에 첨부할 문제 crop 파일명과 검증 손풀이 파일명을 입력한다.
4. 문구 사례를 드롭다운에서 선택만 했을 때 본문이 바뀌지 않는지 확인한 뒤 `선택 문구 적용`을 눌러 본문에 이어 붙는지 확인한다.
5. 중학교/고등학교를 전환해 고등학교에만 `점수 갈림 포인트`가 있는지, 조건부 역할을 끄거나 주요문항을 추가·삭제해도 번호가 1부터 연속인지 확인한다.
6. 입력 누락 슬라이드의 `마스터+상세 복사`와 `전체 복사`가 잠기는지 확인한다. 누락을 채운 뒤 잠금이 풀려야 한다.
7. `프롬프트 작업본 저장`을 누른다. `저장 중 -> 서버 반영 확인 중 -> 저장 완료 · Supabase 재조회 확인` 순서가 보여야 한다.
8. 새로고침 후 입력, 문구 선택, 학교급, 역할 선택, 주요문항 자산 파일명이 그대로인지 확인한다.
9. 마스터와 슬라이드 1개를 복사해 메모장에 붙인다. 한글, 줄바꿈, 페이지 번호, 원본 자산 파일명이 보존되어야 한다. TXT와 JSON도 열어 같은 내용을 확인한다.
10. 실제 GPT Image 호출은 별도 승인 후 문제 crop·손풀이를 직접 첨부해 1장만 시험한다. 자동검증 단계에서는 호출하지 않는다.

## 중단 조건

- 저장 완료가 재조회 전에 표시됨
- 새로고침 후 교사 입력 또는 문구 선택이 사라짐
- 문구 선택만으로 기존 입력이 덮어써짐
- PDF 페이지 정보가 실제 crop 파일로 표시됨
- 누락 슬라이드 복사 잠금이 풀림
- 슬라이드 번호가 끊기거나 중복됨
- 원본 문제/손풀이를 재작성하라는 지시가 프롬프트에 포함됨
- 병렬 리팩터링의 `App.jsx` 충돌 또는 예상하지 못한 shared-file 변경

모든 항목이 기대값과 일치해야 통과다.
