## 공통 구조

1. 인사/시험 소개
   - 학원명, 지점, 학교/학년/고사, 분석 목적을 짧게 시작한다.
   - 일부 글은 바로 한 문단 핵심 요약으로 들어간다.

2. 첫 문단 핵심 요약
   - 시험 전체 성격을 1~3문장으로 정리한다.
   - "점수 갈림 포인트" 같은 별도 추상 라벨은 쓰지 않는다.
   - 대신 기본 문항, 후반부 조건 해석, 시간 관리, 유사교재 연계 같은 시험의 실제 특징을 쓴다.

3. 시험 개요/구조
   - 객관식/서술형/단답형 문항 수, 배점, 만점, 시험범위, 출판사, 등급컷 또는 예상컷을 정리한다.
   - 중등 글은 범위와 문항 분포, 고등 글은 등급컷/예상컷과 부교재 연계가 더 자주 나온다.

4. 출제 흐름/체감 정리
   - 초반부, 중반부, 후반부 흐름을 나누어 설명한다.
   - 난도가 아주 높았는지보다, 어떤 구간에서 조건 해석/그래프 해석/계산량/시간 관리 부담이 생겼는지를 쓴다.

5. 주요문항 반복
   - 문항번호를 중심으로 2~4개를 고른다.
   - 각 문항은 대체로 `문항번호 -> 어떤 유형인지 -> 유사교재/페이지 -> 핵심 개념 -> 학생이 흔들리는 지점 -> 풀이 방향` 순서로 설명한다.
   - 문제 본문이나 정답을 길게 공개하기보다 출제 의도와 접근 방향을 설명한다.

6. 손풀이/이미지 연결
   - 주요문항 설명 근처에 문제 이미지 또는 손풀이 이미지가 들어간다.
   - 앱에서는 자동 crop하지 않고 선생님 crop 이미지를 슬롯에 넣는 구조가 맞다.

7. 다음 학습 방향
   - 다음 단원, 방학, 예비 과정, 학교 학습지 반복, 유사교재 회독, 조건 정리 훈련처럼 실제 학습 행동으로 마무리한다.

8. 수업/특강/상담 연결
   - 학교별 기출 흐름을 수업/보충/특강에 반영한다는 신뢰 문장을 붙인다.
   - 마지막에는 강좌 안내, 상담, 위치, 전화 CTA가 이어진다.

## 앱 체크리스트로 바꿀 때의 기준

- 시험관리의 `시험 후 기록지`는 시험 사실/총평 원본이다.
- 시험분석 산출물 입력은 원본을 다시 쓰는 곳이 아니라 공개글/카드뉴스용 발췌와 보충 메모다.
- `점수 갈림 포인트`처럼 벤치마킹 글에 없는 앱 임의 제목은 전면 라벨에서 제거한다.
- 유지할 입력 묶음은 다음 4개다.
  - 공개글 발췌/보충 메모
  - 블로그 흐름 블록
  - 주요문항 반복 블록
  - 최종 초안 / 복사 / TXT
- 주요문항 반복 블록은 반드시 유지한다. 벤치마킹 글의 핵심은 전체 총평보다 문항번호 중심 설명과 손풀이/이미지 연결이다.

## 최종 체크리스트

1. 인사/시험 소개: 학교, 학년, 고사, 과목, 분석 목적이 있는가?
2. 첫 문단 핵심 요약: 시험 전체 성격이 1~3문장으로 정리되는가?
3. 시험 개요/구조: 문항 구성, 배점, 범위, 등급컷/예상컷이 필요한 만큼 들어가는가?
4. 출제 흐름/체감 정리: 초반/중반/후반 흐름과 체감 난도 이유가 보이는가?
5. 주요문항 선정: 문항번호 2~4개와 선택 이유가 있는가?
6. 주요문항 설명: 유사교재, 핵심 개념, 흔들린 지점, 풀이 방향이 있는가?
7. 손풀이/이미지 슬롯: 문제 crop/손풀이 crop 위치가 명확한가?
8. 다음 학습 방향: 다음 단원, 방학, 예비 과정, 반복 훈련이 구체적인가?
9. 수업/특강/상담 연결: 학원 관리 방식과 CTA가 과장 없이 이어지는가?
10. 최종 초안 보호: AI 재생성이 선생님 수정본을 덮어쓰지 않는가?
