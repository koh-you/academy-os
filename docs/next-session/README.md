```text
이전에는 JS/CSS 렌더러가 Canva를 대체할 수 있는지 검토했고, 실제로 1080x1080 카드뉴스 미리보기, 6개 슬라이드 유형, 주요문항/손풀이 반복 구조, 시험구조 카드의 자동 차트 미리보기까지 구현했습니다. 하지만 현재 디자인 감성은 사용자가 원하는 벤치마킹 카드뉴스 수준에 미치지 못했습니다. 그래서 JS 렌더러는 폐기하지 않되, 최종 제작기 역할은 보류하고 `Canva 재료 생성/구조 검수/차트 생성/장기 자동화 후보`로 역할을 낮춥니다.

현재 결정:
- Canva = 최종 카드뉴스 디자인 편집기
- 웹앱 = 원본 데이터, AI 초안, 선생님 검수본, Canva 재료 export, 블로그/인스타 문구 관리
- JS/CSS 렌더러 = 카드 구조 검수, 문구 길이 확인, 차트/재료 미리보기, 나중에 Puppeteer 자동 PNG 생성 후보
- AI = 블로그 글 문체 정리, 이모티콘/형광펜/줄나눔 규칙 적용, 카드 문구 압축
- 선생님 = 시험 사실 확정, 주요문항 crop, 최종 문구 수정/저장, 공개 승인

반드시 지킬 원칙:
`AI 초안 -> 선생님 검수/수정 -> 선생님 저장본 원본화`

Canva에서 예쁘게 고친 문구가 있으면 웹앱의 선생님 저장본에도 다시 반영해야 합니다. 그렇지 않으면 다음 시험 자동화 때 원본이 갈라집니다.

이모티콘/형광펜 자동 규칙은 카드뉴스 디자인용이 아니라 블로그 글 작성용입니다. 블로그 AI 초안/수정 단계에서 다음 규칙을 적용합니다.
- 인사말 끝: 😊
- 주요문항 시작: 📌
- 핵심 포인트/자주 틀리는 지점: ✅
- CTA/신청 안내: ⬇️⬇️
- 위치: 📍
- 전화번호: ☎
- 형광펜/강조/줄바꿈은 모바일 블로그 가독성 기준으로 적용

다음 작은 작업은 코딩부터 하지 말고, Canva 중심 워크플로우를 화면/데이터 기준으로 재설계하는 것입니다.

우선순위:
1. 현재 Canva 10장 템플릿을 기준으로 각 장의 역할을 확정합니다.
2. 각 카드의 고정 영역과 시험별 교체 영역을 정리합니다.
3. 웹앱이 Canva에 넘겨야 할 재료 목록을 카드별로 정리합니다.
4. 웹앱 산출물 패널의 JS 카드 미리보기 문구를 `최종 카드 제작기`처럼 보이지 않게, `Canva 재료 미리보기/구조 검수`로 재명명할지 검토합니다.
5. 웹앱에서 `Canva 붙여넣기용 텍스트 묶음`, `차트 PNG`, `문제/손풀이 crop 파일명 가이드`, `블로그 초안`, `인스타 캡션`을 export하는 흐름을 설계합니다.
6. 블로그 초안에는 이모티콘/형광펜/줄나눔 규칙을 적용하되, 선생님 저장본을 덮어쓰지 않게 합니다.

주의:
- 10장 기준을 8장 등으로 임의 변경하지 마세요. Canva 페이지 수가 부족하면 Canva 레이아웃 gate로 남깁니다.
- 주요문항 문제/손풀이 이미지는 자동 crop하지 않습니다. 선생님이 직접 crop합니다.
- AI가 문항 수, 시험범위, 문항번호, 정답, 풀이 사실을 추측하면 안 됩니다.
- 필터/fallback으로 화면만 맞추지 말고 원본 row, AI 초안, 선생님 저장본, 산출물 데이터를 구분하세요.
- 테스트 데이터가 꼬이면 보정 덧대기보다 삭제 후 재생성/마이그레이션/재구성 중 무엇이 맞는지 먼저 판단하세요.

작업 완료 시에는 반드시 `사람 검토 절차`와 `AI 자기검토`를 포함하고, 가능한 변경은 검증 후 커밋/푸시까지 진행하세요.
```

## 현재 결론 요약

### 1. 웹앱의 역할

웹앱은 Canva를 대체하는 제작 도구가 아니라 시험분석 산출물의 운영 본부입니다.

- 시험분석 v2 원본과 최종 검수 데이터 보관
- AI 초안 생성 및 선생님 수정/저장 화면 제공
- 선생님 저장본을 이후 산출물의 원본으로 유지
- Canva에 넣을 카드별 문구, 차트 PNG, 파일명/삽입 가이드 생성
- 블로그 초안과 인스타 캡션 관리
- 다음 시험에서도 같은 구조를 반복할 수 있게 원본/재료/검수 상태를 관리

### 2. Canva의 역할

Canva는 최종 카드뉴스 디자인 편집기입니다.

- 카드뉴스의 최종 디자인, 배치, 감성, 업로드용 이미지 생성
- 벤치마킹 카드뉴스처럼 시각 완성도를 빠르게 확보
- 사람이 예외 수정하기 쉬운 최종 편집 공간

단, Canva에서 수정한 문구가 최종 문구가 되면 웹앱의 선생님 저장본에도 반영해야 합니다.

### 3. JS/CSS 렌더러의 현재 역할

폐기하지 않습니다. 다만 최종 카드뉴스 제작기 역할은 보류합니다.

- 카드 구조 검수
- 10장 흐름/주요문항 반복 수 검수
- Canva에 넣을 문구 길이와 데이터 배치 미리보기
- 단원별 출제 비중/난이도 차트 미리보기
- 장기적으로 Puppeteer PNG 자동 생성 후보

현재 사용자가 디자인에 만족하지 않았으므로, JS/CSS를 계속 예쁘게 만드는 쪽으로 밀어붙이지 않습니다.

### 4. 블로그 글 규칙

블로그는 카드뉴스 이미지 사이에 상세 설명글을 붙이는 구조입니다.

- 학부모 대상
- 단원 나열보다 시험 체감, 변별 포인트, 실수 포인트, 학습 전략 중심
- AI가 이모티콘/형광펜/줄나눔을 적용
- 선생님 수정/저장본이 최종 원본

블로그가 답해야 할 질문:
1. 이번 시험이 쉬웠는지 어려웠는지
2. 어디서 점수가 갈렸는지
3. 우리 아이가 틀렸다면 왜 틀렸을 가능성이 큰지
4. 다음 시험까지 무엇을 해야 하는지
5. 이 학원이 학교 시험을 제대로 분석하고 있다는 신뢰감

## Canva 10장 기준

기본 10장 구조는 아래를 기준으로 합니다.

1. 표지
2. 시험 구조
3. 총평 정리
4. 주요문항 1 살펴보기
5. 주요문항 1 손풀이
6. 주요문항 2 살펴보기
7. 주요문항 2 손풀이
8. 주요문항 3 살펴보기
9. 주요문항 3 손풀이
10. 다음 시험 대비 / 블로그 유입 CTA

주요문항이 1~2개인 테스트에서는 해당 카드 쌍만 사용하더라도, 자동화 기준은 10장을 유지합니다.

- 인스타 캡션과 블로그 유입 문구
