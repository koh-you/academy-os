# Solapi 알림톡 템플릿 초안

## 원칙

- 솔라피 템플릿은 자주 바꾸지 않도록 크게 고정한다.
- 세부 항목은 웹앱에서 조건부로 조립해 `#{리포트본문}` 또는 `#{코멘트}`에 넣는다.
- 문장형 안내보다 `항목: 내용` 형식을 기본으로 쓴다.
- 비어 있는 항목은 웹앱에서 제외한다.

## 학부모 알림톡

템플릿 이름: `koh_you_math 학부모 알림톡`

```text
#{학원명}
#{학생명} 학생 수업 안내

#{리포트본문}
```

웹앱이 만드는 `#{리포트본문}` 예시:

```text
출결: 출석
강의 교재: 쎈 공통수학1
강의 내용: 경우의 수
지난 과제: GRIP ~987
다음 과제: 학교 프린트
과제 상태: 풀이 과정을 한 번 더 확인할 필요가 있어 다음 수업에서 점검하겠습니다.
코멘트: 오늘 수업에서 경우의 수 기본 유형을 확인했습니다.
```

조건부 항목:

- `과제 상태`: 학부모용 문장으로 변환해 표시한다.
- `미완료 숙제`: 누적된 미완료 과제가 있을 때만 표시한다.
- `재시험 일정`: 있을 때만 `[중요] 재시험 일정: ...`으로 표시한다.
- `보충 일정`: 있을 때만 `[중요] 보충 일정: ...`으로 표시한다.
- `코멘트`: 강사코멘트가 있을 때만 표시한다.

## 학생 알림톡

템플릿 이름: `koh_you_math 학생 알림톡`

```text
#{학원명}
#{학생명} 학생 안내

#{코멘트}
```

웹앱이 만드는 `#{코멘트}` 예시:

```text
출결: 출석
강의 교재: 쎈 공통수학1
강의 내용: 경우의 수
지난 과제: GRIP ~987
다음 과제: 학교 프린트
코멘트: 오늘 확인한 유형을 다시 정리하고 다음 과제를 진행해 주세요.
```

## 출결 알림톡

템플릿 이름: `koh_you_math 출결 알림톡`

```text
#{학원명}
#{학생명} 학생 출결 안내

#{출결본문}
```

웹앱이 만드는 `#{출결본문}` 예시:

```text
2026-06-17 19:02에 등원 처리되었습니다.
수업: 월수금 7-10반
```

조건부 항목:

- 등원/하원은 지각분과 사유를 표시하지 않는다.
- 지각이면 `지각: 10분`을 표시한다.
- 지각/결석/인정결석이고 사유가 있으면 `사유: ...`을 표시한다.

## Render 환경변수

- `SOLAPI_ATTENDANCE_TEMPLATE_ID`: 출결 알림톡 템플릿 ID
- `SOLAPI_DAILY_REPORT_TEMPLATE_ID`: 학부모 알림톡 템플릿 ID
- `SOLAPI_SPECIAL_LECTURE_TEMPLATE_ID`: 특강 안내 알림톡 템플릿 ID
- `SOLAPI_STUDENT_COMMENT_TEMPLATE_ID`: 학생 알림톡 템플릿 ID

## 특강 안내 알림톡

템플릿 이름: `koh_you_math 특강 안내`

카카오 재검수 권장 본문:

```text
#{학원명} 재원생 보호자님께 드리는 특강 안내입니다.

안녕하세요. #{학원명}입니다.
#{학생명} 학생 보호자님께 특강 일정을 안내드립니다.

특강명: #{특강명}
대상: #{대상}
요일: #{요일}
시간: #{시간}

세부 시수와 수강료, 회차별 일정은 아래 버튼에서 확인해 주세요.
수강을 원하시면 아래 버튼을 눌러 안내문에서 신청해 주세요.
```

버튼:

```text
버튼명: 특강 안내문 보기
버튼 타입: 웹링크
모바일/PC 링크: #{안내문링크}
```

검수 메모:

- 반려 사유가 `수신 대상을 명확하게 확인하기 어려움`이면, 위 본문처럼 `재원생 보호자`라는 수신 맥락을 고정 문장으로 남긴다.
- `#{방학명}`, `#{요일시간}`, `#{시수}`, `#{수강료}`처럼 앱에서 보내지 않는 변수는 쓰지 않는다. 앱이 보내는 변수는 `#{학원명}`, `#{학생명}`, `#{특강명}`, `#{대상}`, `#{요일}`, `#{시간}`, `#{안내문링크}`다.
- 특강명에는 `2026 여름 개별 진도 클리닉`처럼 방학/연도 정보가 들어가므로 별도 `#{방학명}` 변수 없이 재검수한다.
