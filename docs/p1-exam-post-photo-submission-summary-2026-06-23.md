# P1 시험 후 제출 사진 제출 현황과 다음 작업 큐

## 현재 결론

당장은 학생이 시험지를 사진 또는 PDF로 제출하는 방식으로 운영한다.

고속스캐너 자동 분류는 구현하지 않고 보류한다. 다만 나중에 붙일 수 있도록 Storage 경로, 파일 메타데이터, 스캔 배치 테이블 초안은 준비해두었다.

## 지금 구현된 것

- 학생 `오늘` 탭에 시험 후 제출 카드가 표시된다.
- 시험관리 DB의 학교, 학년, 수학 시험일을 기준으로 제출 대상이 자동 계산된다.
- 학생은 점수, 소감, 난이도, 준비도, 잘된 점, 아쉬운 점, 다음 목표/요청사항을 입력할 수 있다.
- 학생은 시험지 사진/PDF를 여러 장 첨부할 수 있다.
- 모바일에서는 파일 선택이 카메라 촬영 흐름으로 이어질 수 있게 `image/*`와 `capture="environment"`를 사용한다.
- 서버는 `/api/exam-post-files`로 파일을 받아 Supabase Storage 비공개 버킷 `exam-submissions`에 저장한다.
- OS에는 파일 원본이 아니라 `bucketId`, `storagePath`, `fileName`, `fileType`, `fileSize`, `source` 메타데이터만 기록한다.
- 강사는 시험관리의 `시험 후 제출` 탭에서 제출/미제출, 점수, 학생 메모, 첨부 파일을 확인하고 확인 완료 처리할 수 있다.
- 파일 업로드가 실패해도 제출 기록은 남고, 강사 화면에 `업로드 실패`로 보인다.

## 당장 운영할 때 확인할 것

1. 학생 계정으로 로그인해서 시험 후 제출 카드가 실제 시험일 이후에만 뜨는지 확인한다.
2. 휴대폰에서 사진 촬영 또는 사진 선택 후 제출한다.
3. 강사 화면 `시험관리 > 시험 후 제출`에서 첨부 파일이 보이는지 확인한다.
4. `파일 보기`가 열리는지 확인한다.
5. 제출 완료 후 학생 화면에서 다시 제출 카드가 `시험 후 제출 완료`로 바뀌는지 확인한다.

## SQL Editor 여부

사진 제출 MVP는 서버가 Storage 버킷을 자동 생성하도록 구현되어 있어, 지금 바로 SQL Editor 실행이 필수는 아니다.

다만 아래 단계로 넘어갈 때는 SQL Editor 실행이 필요하다.

- 파일 메타데이터를 `app_state`가 아니라 정규 테이블로 관리
- 고속스캐너 자동 분류
- 스캔 배치별 검토 화면
- 학생별 파일 히스토리/스토리지 화면

준비된 SQL:

- `supabase/20260623_exam_submission_files.sql`

## 고속스캐너 자동화 보류 메모

자동화 자체는 가능하다. 추천 최종 구조는 다음과 같다.

1. 고속스캐너가 지정 폴더에 PDF/이미지를 저장한다.
2. 로컬 감시 프로그램이 새 파일을 감지한다.
3. 파일명, OCR, 또는 QR 표지로 학생/시험을 추정한다.
4. Supabase Storage에 업로드한다.
5. `exam_submission_files`에 파일 메타데이터와 매칭 상태를 기록한다.
6. Academy OS의 스캔 수신함에서 `자동 매칭`, `검토 필요`, `미매칭`을 확인한다.

가장 안정적인 최종 방식은 QR 표지다. 하지만 당장은 구현하지 않는다.

## 다음 작업 큐

### P0. 사진 제출 운영 검수

- 실제 학생 계정으로 사진 제출 테스트
- 실제 강사 화면에서 파일 열람 테스트
- 업로드 실패 시 학생/강사 화면 문구 확인
- Vercel/Render 배포 후 브라우저 캐시를 비우고 재확인

### P0. 시험 후 제출 데이터 저장 안정화

현재 `examPostSubmissions`는 `app_state` 중심으로 저장된다. 실제 운영에서 제출 데이터가 중요해지면 정규 테이블 저장으로 옮긴다.

- `exam_post_submissions` 테이블 저장 API 추가
- `exam_submission_files` 테이블 저장 API 추가
- 기존 `app_state` 데이터 마이그레이션 또는 병행 읽기
- SQL Editor: `supabase/20260623_exam_submission_files.sql` 실행 필요

### P1. 시험 후 제출 UI 고도화

- 제출 상세 모달 추가
- 파일 여러 장을 썸네일/목록으로 더 보기 좋게 표시
- 미제출 학생만 보기 필터
- 확인 완료/미확인 필터
- 점수 낮음, 난이도 높음, 준비도 낮음 자동 위험 신호 표시
- 강사용 상담 메모 초안 생성

### P1. 시험 후 제출 알림톡

- 시험 다음날 미제출 학생 알림톡 초안 생성
- 제출 완료 학생에게 확인 안내 알림톡 초안 생성
- 실제 발송은 기존 알림톡 안전 스위치와 dry-run 정책을 유지한 뒤 연결

### P1. 학생 오늘 탭 운영 검수

이미 구현했지만 운영 데이터 기준 검수가 필요하다.

- 수업 기록 캘린더가 실제 수업일지 내용을 제대로 보여주는지 확인
- 지난 숙제/다음 숙제 매핑 확인
- 질문 리스트 저장/수정/삭제 확인
- 상단 D-day/특이일정 알림 확인
- 마이페이지 숙제통계/출결통계 확인

### P2. 학부모/상담 연결

- 시험 후 제출 내용을 학부모 리포트나 상담 메모에 요약 반영
- 학생의 아쉬운 이유/도움 요청을 강사 상담 메모 후보로 변환
- 재시험/보충 후보 관리와 연결

### P2. 운영 안정화 큐

이전 md/handoff에 남아 있고 아직 전부 제품화되지 않은 항목이다.

- 알림톡 실제 발송 QA: 학생/학부모 각 1건 테스트, 실패 상태 문구 확인
- 예약 발송 큐가 실제 운영 배치에서 어떻게 소비되는지 확인
- window alert/confirm 의존 UI 정리
- 시험관리 행 삭제/중복 정리/일요보강 삭제 UX 문구 안정화
- 보충관리 통합 저장 후 새로고침 유지 확인
- 자동 생성 수업 후보 패널 접힘 기본값 검토
- 버튼 역할 문구 통일: 저장, 확정, 삭제, 제외, 복구, 중복 삭제
