## 확인된 제작 방식

1. GPT Image 사용 근거
   - 다수 원본 파일명이 `ChatGPT_Image_2026년_...png` 형식으로 남아 있다.
   - 혜성여고는 게시물 이미지 10장 모두, 불암중2와 중계중3도 10장 모두가 `ChatGPT_Image` 파일명이다.
   - 용화여고는 8장 중 6장, 선덕고는 14장 중 12장이 같은 파일명 규칙이다.

2. 반복 수정 근거
   - 혜성여고 10장 세트는 대부분 19:22경 생성됐지만 특정 장은 21:24경 파일로 교체됐다. 같은 게시물 안에서 초기 세트 중 일부 장만 다시 만든 흔적이다.
   - 불암중2 10장 세트는 대부분 12:29경 생성됐고, 특정 두 장이 15:05경 파일로 교체됐다.
   - 용화여고에는 동일한 `03` 번호를 가진 서로 다른 카드가 연속으로 남아 있다. 반복 생성 과정에서 번호/구성 검수가 필요하다는 반례이기도 하다.

3. Canva 고정 템플릿과 다른 점
   - 학교마다 이미지 비율과 디자인 계열이 다르다. 용화여고는 966x1449 세로형 녹색, 혜성여고는 966x1207 보라색, 불암중2·중계중3은 966x966 정사각형 계열이다.
   - 고등학교 카드는 흰 배경·강한 제목·통계형 박스 중심이고, 중학교 카드는 노트/스프링/손그림 장식을 사용한다.
   - 고정된 것은 브랜드 로고 위치, 카드 번호, 큰 제목, 하단 브랜드 띠, 문제→풀이→다음 대비라는 정보 역할이다. 세부 레이아웃은 학교별로 다시 생성된다.

## 반복되는 카드 역할

게시물마다 8~14장으로 차이가 있지만 자주 반복되는 역할은 다음과 같다.

1. 표지/한 줄 총평
2. 시험 구조와 문항 수·배점·범위·등급컷
3. 전체 총평과 변별 포인트
4. 주요문항 원문 + 유사문항/핵심 개념
5. 주요문항 손풀이 + 풀이 핵심/주의점
6. 두 번째 주요문항 원문
7. 두 번째 주요문항 손풀이
8. 필요 시 세 번째 주요문항/손풀이
9. 다음 시험 준비 항목
10. 학원 분석 신뢰/상담·특강 CTA

따라서 Academy OS는 무조건 10장을 즉시 생성하기보다, 기본 10개 역할을 제안하고 선생님이 주요문항 개수에 따라 8~14장 범위에서 장 구성을 확정하는 편이 벤치마크에 더 가깝다.

## 시각적 특징

- 한 장에 하나의 메시지를 크게 제시한다.
- 학교별 포인트 컬러를 전 카드에 유지한다.
- 표지·시험 구조·다음 대비 카드는 GPT Image 통렌더에 가깝다.
- 주요문항 카드는 시험지 원문 이미지와 손풀이 이미지를 큰 영역에 넣고, GPT Image가 제목·테두리·요약 박스·아이콘을 조립한 하이브리드다.
- 문제 카드에는 `출제 단원`, `핵심 개념`, `변별 포인트`, `유사문항 출처`가 반복된다.
- 풀이 카드에는 `풀이 핵심 3줄`, `문제풀이 TIP`, `주의 포인트`, 한 줄 판단이 반복된다.
- 블로그 본문은 카드 이미지만 나열하지 않는다. 인사→시험 총평→구조→주요문항 해설→다음 대비→학원 신뢰 문장→CTA와 지점 정보 순으로 장문 설명을 붙인다.

## 확인된 위험과 사람 gate

- GPT Image가 카드 번호를 중복하거나 순서를 틀릴 수 있다.
- 긴 한국어 문장, 수학식, 문제 원문을 이미지 생성에 맡기면 오탈자와 수식 변형 위험이 있다.
- 시험문제와 손풀이는 생성시키지 말고 선생님이 확인한 원본 이미지를 사용해야 한다.
- 로고와 학원명은 변형될 수 있으므로 공식 원본을 reference로 고정하거나 생성 후 결정론적으로 합성하는 방식을 검토한다.
- 각 장은 `초안 생성 -> 선생님 검토 -> 수정 요청 -> 이전 이미지 기반 재생성 -> 최종 선택` 단계를 가져야 한다.
- 최종 선택 전 번호, 학교명, 학년/고사, 문항번호, 점수, 등급컷, 수학식, 로고, CTA를 사람이 확인해야 한다.

## Academy OS 구현 방향

- Canva/미리캔버스 프로젝트가 원본이 아니라 `GPT Image 생성 세트 + 장별 버전 이력 + 선생님 최종 선택본`을 원본으로 둔다.
- 분석 run과 이미지 세트를 연결하고, 각 장마다 역할·원천 snapshot·생성 프롬프트·수정 요청·부모 버전·Storage 경로·최종 선택 여부를 저장한다.
- 유료 호출은 장별 명시적 버튼으로만 실행하며 10장 자동 일괄 생성은 하지 않는다.
- 분석 데이터가 바뀌어도 기존 생성본을 자동 덮어쓰지 않고 `원천 변경됨` 경고 후 선생님이 재생성 여부를 선택한다.
- 최종 내보내기는 선택된 장만 번호순으로 묶는다. Canva/미리캔버스가 필요하면 최종 선택 이미지의 후처리/배치 수단으로만 둔다.
