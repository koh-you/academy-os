## 매우 중요: 자동배포 운영 원칙

앞으로 Codex 세션에서 코드를 수정한 경우 반드시 아래 순서를 끝까지 수행한다.

1. 변경 범위 확인: `git diff`, `git status --short`
2. 검증 실행: 최소 `npm run build`
3. 운영 영향 변경이면 `npm run test:production`도 실행
4. 커밋 생성
5. GitHub push
6. 자동배포 상태 확인
   - Frontend: Vercel 배포가 최신 커밋으로 성공했는지 확인
   - Backend/API/cron 관련 변경: Render 배포 또는 cron 실행 상태 확인
7. 사용자가 “배포됐나요?”라고 다시 묻지 않아도 Codex가 먼저 확인하고 보고한다.

주의:

- GitHub에 push했다고 끝난 것이 아니다.
- Vercel/Render 자동배포 완료까지 확인해야 실제 운영 반영이 끝난 것이다.
- 배포 지연이 있으면 “코드는 push됐고, Vercel/Render 반영 대기 중”처럼 상태를 구분해서 보고한다.
- 배포 실패가 보이면 로그를 확인하고 가능한 수정까지 진행한다.
- 이 원칙은 매 대화 세션마다 반복해서 지켜야 한다.
