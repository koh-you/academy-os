# 시험 후 제출함 학생 웹앱 구현 계획

## 목적

Tally로 따로 취합하던 시험 후 업무를 Academy OS 학생 웹앱 안으로 가져온다.

대상 업무:

- 시험 점수 제출
- 시험 후 셀프체크리스트 제출
- 학생 메모 제출
- 시험지 사진 또는 PDF 제출
- 강사 화면에서 시험별 제출 현황 확인

재시험/보충 후보 관리는 이미 별도 탭이 있으므로, 이 문서의 범위에서는 새 후보 탭을 만들지 않는다. 시험 후 제출 데이터에서 필요한 경우 기존 재시험/보충 관리 흐름으로 연결만 한다.

## 권장 UX

학생 웹앱에 시험 후 제출함을 항상 크게 띄우지 않는다.

권장 방식:

```text
평소
→ 홈에는 노출하지 않거나, 더보기/작은 메뉴에 "시험 제출함"만 둔다.

시험 이후
→ 학생 홈 상단에 "시험 후 제출 필요" 카드 자동 노출
→ 점수 입력 / 셀프체크 / 시험지 제출 버튼 표시

제출 완료 후
→ 홈 카드는 숨김
→ 제출함 메뉴에서는 제출 이력으로 확인 가능
```

기본 오픈 규칙:

```text
시험일 당일 15:00 오픈
마감: 시험 다음날 23:59
미제출이면 홈 카드 계속 표시
제출 완료하면 홈 카드 숨김
```

학생별 시험 일정은 시험관리의 학교/학년/과목/시험일 데이터를 기준으로 자동 연결한다.

예시:

```text
창동고 1학년 공통수학1 시험일: 2026-06-25
→ 2026-06-25 15:00 학생앱 제출함 오픈
→ 2026-06-26 23:59 마감
→ 미제출 학생은 강사 화면에 미제출 표시
```

## 학생 화면

학생 홈 카드 예시:

```text
시험 후 제출 필요

창동고 1학년 공통수학1 기말고사
마감: 6월 26일 23:59

[점수 입력] [셀프체크] [시험지 제출]
```

학생 제출 항목:

- 이름/학년은 로그인 학생 정보에서 자동 입력한다.
- 시험명, 학교, 학년, 과목, 시험일은 시험관리 데이터에서 자동 연결한다.
- 제출 시각은 서버가 자동 기록한다.
- 시험 점수
- 셀프체크리스트
- 학생 메모
- 시험지 사진/PDF

셀프체크리스트 예시:

- 시간이 부족했다.
- 계산 실수가 많았다.
- 개념이 부족했다.
- 문제 해석이 어려웠다.
- 시험범위 준비가 부족했다.
- 서술형/풀이 과정에서 감점이 있었다.
- 마지막 검토 시간이 부족했다.

학생 메모 예시:

```text
마지막 두 문제를 시간이 부족해서 못 풀었습니다.
서술형에서 풀이 과정을 어떻게 써야 할지 헷갈렸습니다.
```

## 강사 화면

시험관리 안에 `시험 후 제출 관리` 화면을 둔다.

추천 구조:

```text
시험관리
├ 시험 일정
├ 시험 대비
├ 시험 후 제출 관리
└ 기존 재시험/보충 후보 탭
```

시험 후 제출 관리 화면은 학교/학년/과목/시험별로 묶어서 보여준다.

요약 카드:

```text
2026 1학기 기말 · 창동고 1학년 · 공통수학1

대상 8명
제출 완료 6명
미제출 2명
확인 완료 4명
```

학생 목록:

```text
김OO  86점    셀프체크 완료  시험지 제출  확인 전
이OO  미제출  미제출        미제출      알림 필요
박OO  72점    셀프체크 완료  시험지 제출  확인 완료
```

학생 상세:

```text
김OO

점수
- 86점

셀프체크
- 계산 실수 많음
- 시간 부족
- 서술형 감점

학생 메모
- 마지막 문제 시간이 부족했습니다.

제출 파일
- 시험지 사진 4장
- 오답 표시 사진 1장

강사 확인
- 확인 전 / 확인 완료
```

## 관리 기능

필수:

- 시험별 제출 현황 요약
- 학생별 제출/미제출 상태
- 점수 확인
- 셀프체크 답변 확인
- 시험지 파일 보기
- 강사 확인 완료 체크
- 미제출 학생 필터

있으면 좋은 기능:

- 점수 낮은 학생 필터
- 셀프체크 위험 신호 필터
  - 시간 부족
  - 계산 실수
  - 개념 부족
  - 시험범위 누락
- 미제출 학생 알림톡 초안 생성
- 제출 내용 학부모 알림톡/상담 메모에 반영

재시험/보충 후보는 별도 탭이 있으므로 이 화면에서는 깊게 다루지 않는다. 필요한 경우 학생 상세에서 기존 재시험/보충 관리로 이동하는 링크만 둔다.

## Tally 폼 기준 자동화 매핑

기존 Tally 폼: `https://tally.so/r/D40XWq`

CSV 확인 기준:

- 제출 데이터 24건
- 컬럼 22개
- 주요 질문: 이름, 학년, 시험 전반 소감, 난이도, 준비 충분도, 잘된 점, 아쉬운 이유, 공부 과정 어려움, 수업/자료 도움 여부, 다음 시험 목표, 다음 시험에서 바꿀 점, 선생님께 도움받고 싶은 부분, 자유 건의

### 자동화할 항목

학생이 직접 입력하지 않아도 되는 항목:

- 이름: 로그인한 학생 계정의 `students.name`
- 학년: `students.grade`
- 학교: `students.schoolName`
- 시험 구분: 현재 선택된 `exam_prep_rows.examCycle`
- 과목: `exam_prep_rows.subject`
- 시험일: `exam_prep_rows.mathExamDate` 또는 학사일정 수학시험 날짜
- 제출 대상 여부: 시험일 당일 15:00 이후 자동 오픈
- 제출 마감: 시험 다음날 23:59 자동 계산
- 제출 시각: 서버 저장 시 `submitted_at`
- 미제출 상태: 마감 전/후 자동 판정
- 강사 확인 전/완료 상태: 강사 화면에서 체크

### 학생이 입력할 항목

Tally 질문 중 학생의 자기평가가 필요한 항목:

- 이번 시험, 전체적으로 어떻게 느꼈나요?
  - 기대보다 잘 봤다

  - 기대에 못 미쳤다
  - 모르겠다
- 시험 난이도는 어느 정도였나요? `0~10`
- 시험 준비가 충분했다고 느끼나요? `0~10`
- 스스로 잘 준비했다고 느낀 부분
- 실력을 발휘할 수 있었던 문제 유형/단원
- 아쉬웠던 이유
  - 아는데 실수했다
  - 준비가 부족했다
  - 시간이 부족했다
  - 멘탈이 흔들렸다
  - 컨디션 문제
  - 기타
- 더 준비할걸 했던 부분
- 시험장에서 가장 아쉬웠던 순간
- 수학 공부과정에서 가장 힘들었던 것
  - 개념이 헷갈린다
  - 문제가 안 풀린다
  - 공부 의지가 안 생긴다
  - 시간이 없다
  - 기타
- 학원 수업과 자료가 도움이 됐나요?
  - 많이 도움됐다
  - 어느 정도 됐다
  - 잘 모르겠다
  - 별로 안 됐다
- 수업이나 자료에서 좋았던 점/아쉬운 점
- 다음 시험 목표 점수 또는 등급
- 다음 시험을 위해 가장 바꾸고 싶은 것 한 가지
- 선생님께 꼭 도움받고 싶은 부분
- 선생님한테 하고 싶은 말/건의사항

### 강사 화면 자동 요약 후보

학생 제출 후 강사 화면에는 다음처럼 자동 요약한다.

```text
학생: 박OO
시험: 창동고 1학년 공통수학1 기말
점수/목표: 86점 / 다음 시험 1등급
위험 신호: 시간 부족, 계산 실수
도움 요청: 문제 빨리 푸는 법
강사 확인: 확인 전
```

자동 위험 신호 예시:

- `준비 충분도 <= 5`: 준비 부족 경고
- `난이도 >= 8`: 난도 체감 높음
- 아쉬웠던 이유에 `시간이 부족했다`: 시간관리 지도 필요
- 아쉬웠던 이유에 `아는데 실수했다`: 계산/검토 루틴 필요
- 공부 과정 어려움에 `개념이 헷갈린다`: 개념 보강 필요
- 공부 과정 어려움에 `문제가 안 풀린다`: 유형 접근 훈련 필요

### MVP 범위 추천

1단계에서는 시험지 파일 업로드보다 자기평가 입력과 강사 요약을 먼저 구현한다.

- 자동 입력: 학생명/학년/학교/시험/과목/마감
- 학생 입력: 점수, 소감, 난이도, 준비도, 체크리스트, 목표, 도움 요청
- 강사 확인: 학생별 제출/미제출, 위험 신호, 확인 완료

2단계에서 시험지 사진/PDF 업로드를 붙인다.

## 알림 흐름

학생 웹앱 홈 카드가 기본 알림 역할을 한다.

추가로 필요한 경우:

```text
제출함 오픈
→ 학생 홈 카드 표시
→ 마감 전 미제출 학생에게 학생 알림톡 또는 웹앱 알림

```

알림톡 예시:

```text
[으뜸수학 고태영T] 학생 알림톡

김OO 학생, 시험 후 제출함 작성이 필요합니다.

시험: 창동고 1학년 공통수학1 기말고사
제출 항목: 점수, 셀프체크, 시험지 사진
마감: 6월 26일 23:59
```

## 구현 단계

### 1단계: 제출함 MVP

- 시험관리 데이터 기준으로 학생별 제출 대상 생성
- 학생 웹앱 홈에 오픈된 제출 카드 표시
- 점수 입력
- 셀프체크 제출
- 학생 메모 제출
- 강사 화면에서 제출/미제출/확인 완료 상태 확인

### 2단계: 시험지 업로드

- 학생 웹앱에서 사진/PDF 업로드
- Supabase Storage 연동
- 강사 화면에서 파일 미리보기 또는 다운로드
- 파일 제출 여부 표시

### 3단계: 알림 연결

- 마감 전 미제출 알림톡 초안 생성
- 웹앱 홈 카드 유지
- 강사 화면에서 미제출 학생만 필터링

### 4단계: 기존 흐름 연결

- 제출 상세에서 기존 재시험/보충 후보 탭으로 이동
- 학부모 알림톡 또는 상담 메모에 시험 후 제출 내용 반영

## 구현 시 주의사항

- 시험 후 제출함은 시험 기간 외에는 학생 홈을 방해하지 않게 조건부 노출한다.
- 학생이 같은 시험에 중복 제출하지 않도록 `exam_prep_id + student_id` 기준으로 한 제출 레코드를 유지한다.
- 수정 제출은 허용하되, `submitted_at`, `updated_at`을 구분한다.
- 파일 업로드는 용량 제한과 파일 형식을 제한한다.
- 시험지 파일에는 개인정보와 학교 시험지가 포함될 수 있으므로 공개 URL을 피하고 권한 있는 경로로 관리한다.
- 재시험/보충 후보 관리는 기존 탭을 유지한다.

## 최종 방향

```text
Tally로 받던 시험 후 점수/셀프체크/시험지 제출
→ 학생 웹앱 제출함으로 이전
→ 선생님은 시험관리 안에서 자동 취합 확인
→ 재시험/보충은 기존 별도 탭에서 계속 관리
```
