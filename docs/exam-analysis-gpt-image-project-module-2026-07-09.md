# 시험분석 GPT Image 프로젝트 모듈 구조 - 2026-07-09

## 목표

시험분석 총평/체크리스트에 작성한 내용을 GPT 프로젝트 대화세션에 넣고, GPT image로 카드뉴스 슬라이드를 한 장씩 반복 생성한다.

핵심 목표는 다음 세 가지다.

- 벤치마킹 블로그의 전체 구성과 정보 배치 방식은 참고하되, 문장/디자인/브랜드는 복제하지 않는다.
- 선생님이 체크리스트에서 확정한 시험 사실을 원본으로 삼는다.
- GPT image는 원본 데이터를 카드뉴스 이미지로 렌더링하는 도구이며, 시험 사실을 새로 추측하지 않는다.

## 반복 제작 구조

```text
웹앱 기존 시험 데이터
-> 시험지분석 확정/후보 데이터
-> 선생님 확인 체크리스트
-> 시험 특징 분류
-> 10장 카드뉴스 기획안
-> 카드별 GPT image 생성 세션
-> 카드별 수정 세션
-> 최종 카드뉴스/블로그/인스타 반영
```

## 데이터 원본 레이어

### 1. webAppExamSeed

웹앱에 이미 저장된 시험관리/총평 데이터다. 체크리스트를 매번 빈칸에서 시작하지 않도록 기본값으로 쓴다.

- 학교/학년/고사
- 시험일
- 시험범위
- 교과서/출판사
- 기존 시험 후 총평/체크리스트
- 수업/보충/상담 연결 메모

### 2. examAnalysisSeed

시험지분석 v2에서 가져오는 구조 데이터다. 선생님 검수 전에는 후보이며, 검수/확정 후에만 공개 산출물 원본으로 쓴다.

- 객관식/서술형/단답형 수
- 총점/배점
- 단원별 실제 비중
- 문항별 단원/유형/난도 분포
- 초반/중반/후반 난도 흐름 후보
- 학생들이 흔들릴 지점 후보
- 주요문항 후보

### 3. teacherConfirmChecklist

선생님이 확인/수정/확정하는 제작 원본이다. GPT image와 블로그/인스타 산출물은 이 레이어 이후의 값만 사용한다.

- 등급컷/예상 등급컷
- 부교재/모의고사 출제 근거
- 주요문항 최종 선택
- 주요문항별 유사유형 근거
- 다음 대비
- 카드뉴스 톤
- 쓰면 안 되는 교재명/문항번호/불확실 정보

### 4. featureClassification

시험지분석 결과, Vision AI 분석, 키워드 사전을 참고해 분류한 값이다. 선생님 검수 전에는 후보이며, 검수 후에만 카드뉴스 원본으로 쓴다.

- 출제 원천
- 시험 구조
- 단원 비중
- 난도 흐름
- 학교별 특징
- 주요문항 패턴
- 다음 학습 방향

### 5. cardPlan

이미지를 만들기 전 확정하는 10장 카드 기획안이다.

- 카드 번호
- 카드 역할
- 메인 문구
- 보조 문구
- 강조 키워드
- 반드시 넣을 숫자/표
- 검수 필요 사실

### 6. cardRenderPacket

GPT image에 카드 1장씩 넘기는 입력이다. 각 대화세션이 이전 대화를 몰라도 생성 가능하도록 브랜드 규칙, 학교 특징, 해당 카드 정보를 함께 넣는다.

### 7. teacherFinal

생성 이미지와 문구를 선생님이 검수한 최종본이다. 이후 블로그/인스타/Canva/다운로드의 원본이다.

## GPT 프로젝트 고정 지침

GPT 프로젝트 Instructions에 아래 내용을 넣는다.

```text
너는 으뜸수학 고태영T의 시험분석 카드뉴스 제작 어시스턴트다.

역할:
- 선생님이 확정한 시험 사실을 바탕으로 블로그/인스타용 카드뉴스를 기획하고 GPT image용 카드별 프롬프트를 만든다.
- 필요하면 GPT image로 1080x1350 세로 카드뉴스 이미지를 한 장씩 생성한다.
- 벤치마킹 블로그의 정보 구성 방식은 참고하되, 문장/색상/브랜드/레이아웃을 그대로 복제하지 않는다.

브랜드:
- 브랜드명은 "으뜸수학 고태영T"다.
- 톤은 블루/화이트 중심, 전문적이고 차분한 수학 내신 분석 스타일이다.
- 학부모가 보았을 때 시험을 꼼꼼히 분석하고 다음 대비까지 관리한다는 인상이 나야 한다.

원본 원칙:
- 선생님이 제공한 체크리스트/확정값만 시험 사실로 사용한다.
- 시험범위, 문항번호, 교재명, 페이지, 배점, 등급컷은 추측하지 않는다.
- 객관식/서술형/단답형 수와 단원별 실제 비중은 Academy OS 시험지분석 결과 또는 선생님 확정값만 사용한다.
- 주요문항은 AI가 임의로 고르지 않는다. 시험지분석은 후보를 낼 수 있지만, 카드뉴스에 들어갈 주요문항 최종 선택은 선생님이 한다.
- 불확실한 정보는 카드에 넣지 말고 "검수 필요"로 남긴다.
- AI가 만든 문구는 초안이며, 선생님이 수정/확정한 값이 최종 원본이다.

디자인 원칙:
- 카드뉴스는 1080x1350 세로 비율을 기본으로 한다.
- 한 카드에는 하나의 역할만 둔다.
- 텍스트는 모바일에서 읽히도록 짧게 쓴다.
- 숫자, 문항 수, 등급컷, 교재명, 주요 키워드는 배지/표/강조 블록으로 정리한다.
- 장식보다 정보 구조를 우선한다.

금지:
- 원본 블로그 문장 장문 복제 금지
- 원본 브랜드명 사용 금지
- 원본 색상/장식/레이아웃 그대로 복제 금지
- 제공되지 않은 시험 사실 추가 금지
- 너무 많은 텍스트를 한 카드에 넣기 금지
```

## GPT 프로젝트 대화세션 구조

### 세션 1. 시험별 기획 세션

대화 제목 예시:

```text
불암중2 2026 1학기 기말 카드뉴스 기획
```

첫 메시지에 아래 패킷을 붙여 넣는다.

```text
[작업 목표]
아래 체크리스트를 바탕으로 10장 카드뉴스 기획안을 만들어줘.
아직 이미지를 만들지 말고, 카드별 내용만 표로 정리해줘.

[브랜드]
- 으뜸수학 고태영T
- 블루/화이트 중심
- 전문적이고 차분한 내신 분석 카드뉴스
- 원본 벤치마킹 블로그의 문장/디자인 복제 금지

[체크리스트]
{선생님이 작성한 체크리스트 전체 붙여넣기}

[출력 형식]
카드번호 / 카드 역할 / 메인 문구 / 보조 문구 / 강조 키워드 / 반드시 넣을 숫자/표 / 검수 필요 사실

[10장 기본 구성]
1. 표지
2. 시험 구조
3. 단원별 출제 비중
4. 난도 흐름
5. 출제 원천
6. 주요문항 1
7. 주요문항 2
8. 학생들이 흔들린 지점
9. 다음 대비 전략
10. 으뜸수학 고태영T 관리 방식/CTA

[주의]
- 체크리스트에 없는 시험 사실은 추가하지 마.
- 주요문항은 `주요문항 최종 선택`에 있는 문항만 사용해.
- 불확실한 내용은 카드에 넣지 말고 검수 필요 사실에 적어.
- 한 카드에 문구를 너무 많이 넣지 마.
```

세션 1의 결과는 `CONFIRMED_CARD_PLAN`으로 저장한다. 선생님이 수정한 뒤 확정된 카드 기획안만 다음 세션에서 사용한다.

### 세션 2~11. 카드별 생성 세션

각 카드마다 별도 대화세션을 만들거나, 같은 세션에서 카드 번호를 나누어 진행한다. 안정성을 위해 카드 1장당 1회 생성 요청을 기본으로 둔다.

대화 제목 예시:

```text
불암중2 카드 01 표지 생성
불암중2 카드 02 시험 구조 생성
```

각 카드 세션에 아래 패킷을 붙여 넣는다.

```text
[작업 목표]
아래 정보만 사용해서 1080x1350 세로 카드뉴스 이미지 1장을 만들어줘.

[브랜드/스타일]
- 으뜸수학 고태영T
- 블루/화이트 중심
- 전문적이고 차분한 내신 분석 스타일
- 숫자와 키워드가 한눈에 보이는 카드뉴스
- 벤치마킹 블로그의 정보 구조는 참고하되 문장/디자인/브랜드는 복제하지 않음

[학교 시험 특징 요약]
- 학교/학년/고사:
- 출제 원천:
- 시험 구조:
- 단원 비중:
- 난도 흐름:
- 학교별 특징:
- 주요문항 후보 패턴:
- 주요문항 최종 선택:
- 다음 대비:

[이번 카드]
- 카드 번호:
- 카드 역할:
- 메인 문구:
- 보조 문구:
- 강조 키워드:
- 반드시 넣을 숫자/표:

[레이아웃 지시]
- 상단에 학교명/학년/고사명을 작게 배치
- 중앙에 메인 문구를 가장 크게 배치
- 핵심 숫자/키워드는 배지 또는 간단한 표로 표시
- 하단에 "으뜸수학 고태영T"를 작게 표시
- 한글 텍스트가 잘리지 않게 여백 확보
- 모바일에서 읽히는 크기 유지

[금지]
- 제공되지 않은 시험 사실 추가 금지
- 주요문항 후보 패턴을 선생님 최종 선택처럼 사용 금지
- 원본 블로그 문장/디자인 복제 금지
- 원본 브랜드명 사용 금지
- 너무 작은 글씨 금지
```

## 카드 수정 세션 프롬프트

생성된 이미지가 마음에 들지 않을 때는 같은 세션에서 아래처럼 수정한다.

```text
방금 생성한 카드 이미지를 수정해줘.

[유지할 것]
- 브랜드: 으뜸수학 고태영T
- 블루/화이트 톤
- 카드 역할:
- 반드시 유지할 시험 사실:

[수정할 것]
- 바꿀 문구:
- 줄일 문구:
- 키울 텍스트:
- 줄일 텍스트:
- 강조할 키워드:
- 제거할 요소:
- 여백/가독성 요청:

[금지]
- 새로운 시험 사실 추가 금지
- 카드 역할 변경 금지
- 원본 벤치마킹 디자인으로 되돌리기 금지
```

## 카드별 기본 역할

| 카드 | 역할 | 목적 | 입력해야 할 핵심 정보 |

## 검수 Gate

이미지를 다음 단계로 넘기기 전 아래를 확인한다.

- 학교명/학년/고사명이 맞는가?
- 문항 수/배점/등급컷이 체크리스트와 같은가?
- 교재명/부교재/문항번호가 체크리스트에 있는 값인가?
- 주요문항이 선생님 최종 선택 문항과 같은가?
- 한 카드의 정보량이 너무 많지 않은가?
- 모바일에서 읽히는가?
- 원본 블로그 브랜드나 문장을 복제하지 않았는가?
- `으뜸수학 고태영T` 브랜드가 들어갔는가?

## 중단 조건

- GPT image가 없는 시험 사실을 추가함
- 문항번호/등급컷/교재명을 틀리게 씀
- 한글 텍스트가 잘리거나 깨짐
- 원본 벤치마킹 디자인과 너무 비슷함
- 카드 역할이 섞임
- 선생님 확정 전 AI 문구를 최종본처럼 사용함

## 이후 Academy OS 연결 방향

나중에 웹앱에 붙일 때는 아래 흐름으로 만든다.

```text
웹앱 시험관리/총평 데이터 자동 입력
-> 시험지분석 확정/후보 데이터 자동 입력
-> 선생님 확인 체크리스트 편집
-> 선생님 저장
-> 10장 카드 기획안 AI 생성
-> 선생님 카드 기획안 수정/확정
-> 카드별 GPT image 프롬프트 생성
-> 이미지 생성/업로드
-> 카드별 수정/확정
-> 블로그/인스타/다운로드 산출물 반영
```

저장 원천은 `webAppExamSeed`, `examAnalysisSeed`, `teacherConfirmChecklist`, `confirmedCardPlan`, `finalCardImages`로 분리한다. 웹앱에는 항목별 출처 배지를 표시한다: `웹앱 자동 입력`, `시험지분석 후보`, `선생님 확정 필요`, `확정 완료`, `생성 금지`. AI 초안이 선생님 확정본을 덮어쓰면 안 된다.
