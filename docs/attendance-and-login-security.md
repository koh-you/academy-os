# 출결 전용 화면과 로그인 보안 정리

## 출결 전용 화면

전용 주소:

```text
/attendance
```

운영 URL 예시:

```text
https://academy-os-blue.vercel.app/attendance
```

이 주소에서는 로그인 화면, 좌측 메뉴, 학생/학부모/강사 화면이 보이지 않고 출결 체크 화면만 표시한다.

## 지각 기준

기본 기준:

- 등원 기준 시간은 수업 정각이다.
- 수업 시작 시간 이후 등원하면 자동으로 `지각` 처리한다.

운영 보정:

- 학교 일정 때문에 늦는 경우는 강사가 수업일지에서 `출석` 또는 `인정 지각` 성격으로 수정하고 사유를 남긴다.
- MVP에서는 자동 판정은 단순하게 유지하고, 예외는 강사가 보정한다.

다음 단계 후보:

- 학생별 허용 지각 시간
- 학교별 행사/시험기간 예외
- 특정 날짜만 지각 기준 완화
- 지각 알림톡에 사유가 있을 때만 조건부 표시

## 로그인 보안

현재 반영:

- 공개 로그인 화면에서 테스트 계정 힌트를 제거한다.
- 학생은 학생별 `loginId + PIN`으로 로그인할 수 있다.
- 학부모는 `parent-학생loginId + PIN`으로 로그인할 수 있다.
- 테스트용 `student / 1234`, `parent / 1234`, `teacher / 1234`는 개발 검수용으로만 남겨둔다.

실제 운영 전 필수 작업:

- Supabase Auth 또는 서버 세션 기반 로그인으로 교체
- 비밀번호/PIN 해시 저장
- 학생/학부모/강사 권한 서버 검증
- 로그인 실패 횟수 제한
- 테스트 계정 비활성화
- HTTPS 배포 주소만 사용
