# Day 26-30 MVP Closeout

## MVP Scope

월수금 7-10반 8명을 기준으로 수업일지 중심 MVP를 마감한다. 데이터 원천은 노션 학생관리 페이지와 노션 수업 DB에서 확인한 실제 월수금 7-10반 데이터이며, 확인되지 않은 날짜와 학생별 수업 행은 임의 생성하지 않는다.

## Included

- 역할별 로그인
  - 학생: `student / 1234`
  - 학부모: `parent / 1234`
  - 선생님: `teacher / 1234`
- 선생님 수업일지
  - 월간 달력
  - 수업 클릭 시 별도 수업일지 화면 진입
  - 학생 8명 전체 표시
  - 출결 체크 팝업
  - 지각/결석 시간 및 사유 입력
  - 지난 숙제 직접 입력
  - 다음 숙제 직접 입력
  - 전체 숙제 일괄 적용 후 학생별 개별 수정
  - 미완료 숙제 메모
  - 강사코멘트
  - 학생코멘트
- 숙제 현황
  - 미완료/미검사 숙제 확인
- 학생 관리
  - 월수금 7-10반 학생 기본 정보 관리
- 평가/시험 관리
  - 교재오답 추적
  - 내신/모의고사 성적 관리
  - 학원 테스트 관리

## Not Included Yet

- 실제 문자/카카오/알림 발송
- 실제 삭제 동기화
- 노션 양방향 자동 동기화
- BTLMath 계정과 실시간 데이터 연동
- 전체 반 확장 자동화
- AI API 실제 보고서 생성

## Data Source Notes

현재 앱의 로컬 데이터는 노션에서 확인한 실제 월수금 7-10반 행을 코드에 반영한 것이다. 노션과 실시간으로 연결된 상태는 아니다. 추후 Day 31 이후에는 노션 API 또는 별도 백엔드 동기화 계층을 두고, `StudentId`, `LessonId`, `HomeworkId`, `ProblemId`, `ExamId`, `ReportId`를 공통 ID로 고정해야 한다.

## Day 31 Suggested Work

- 노션 수업 DB에서 월수금 7-10반 데이터를 자동으로 가져오는 import 스크립트 작성
- 수업 DB 행과 앱 내부 `LessonStudentRecord`의 매핑 규칙 확정
- 학생 DB의 학교/학년/학부모 정보 rollup 처리 방식 확정
- 저장 실패 전역 배너와 변경 이력 로그 추가
- 전체 반 확장을 위한 ClassTemplate 기반 필터링 추가
