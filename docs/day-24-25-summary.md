# Day 24-25 Summary

## Goal

학생, 학부모, 강사가 서로 다른 계정으로 입장하고 역할에 따라 다른 화면을 보게 만든다. 같은 숙제에서 발생한 후속조치는 중복으로 새로 만들지 않고, 하나의 원본 숙제를 중심으로 계속 추적하면서 보충 숙제를 다시 낼 수 있게 한다.

## Completed

- 로그인 화면을 학생, 학부모, 선생님 역할 탭으로 분리했다.
- 데모 계정은 요청대로 단순 고정 계정으로 맞췄다.
- 학생 로그인은 학생 본인 화면만 보이도록 제한했다.
- 학부모 로그인은 학생 1명의 데이터를 열람하는 읽기 전용 화면으로 연결했다.
- 선생님 로그인은 기존 운영 화면으로 연결했다.
- 학부모 화면에는 알림, 보고서, 숙제, 출결, 커리큘럼 탭을 두었다.
- 학부모 화면에서는 숙제 재분배, 수정, 삭제, 발송 같은 조작 버튼을 제거했다.
- 후속조치 생성 시 `studentId + sourceId + taskType`이 같은 항목은 새로 만들지 않고 기존 항목을 계속 추적한다.
- 후속조치 카드에서 `숙제 내기`를 누르면 보충 숙제가 생성되고, 원본 후속조치에 `lastHomeworkId`, `childHomeworkIds`, `attemptCount`가 누적된다.
- 실제 발송, 실제 외부 인증, 실제 삭제는 아직 구현하지 않았다.

### Korea Date Rule

화면 날짜와 새로 생성되는 보충 숙제 날짜는 `Asia/Seoul` 기준으로 계산한다. 로컬 컴퓨터 시간이 다른 나라 시간대여도 앱의 오늘 날짜는 한국 날짜를 기준으로 한다.

## Guardrails

- 선생님 로그인만 운영 메뉴를 볼 수 있다.
- 학생 로그인은 필터링된 학생 배열만 받으므로 다른 학생 데이터로 전환할 수 없다.
- 학부모 화면은 읽기 전용이다.
- 후속조치는 같은 원본 숙제를 계속 추적한다.
- 알림은 모의 로그만 남기고 실제 발송하지 않는다.
- 브라우저에 남아 있던 이전 샘플 데이터가 새 반 데이터를 가리지 않도록 로컬 저장소 키를 갱신했다.
