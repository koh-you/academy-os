## 2026-06-02 Day 1 Follow-up

### Clarified Concepts

- `요구사항 7개를 도메인에 매핑한다`는 말은 각 요구사항을 어느 데이터 묶음에 저장하고 어떤 공통 ID로 연결할지 정한다는 뜻이다.
- 예: 학생 화면의 숙제 `완료 체크`는 화면 기능이지만 데이터상으로는 `HomeworkSubmission`에 저장되고 `StudentId`, `HomeworkId`로 연결된다.

### Decisions Added

- 첫 파일럿 반은 `월수금 4-7반 (고1 내신)`으로 정한다.
- 운영 용어는 수업, 보강, 재시험, 숙제보충을 그대로 사용한다.
- 강사 화면은 날짜 -> 해당 일자의 수업 목록 -> 선택한 수업의 반 학생 목록 순서로 이동한다.
- 하루에 수업이 2개 있으면 달력/목록에 수업 2개가 보이고, 강사는 그중 하나를 클릭해 해당 반 화면으로 들어간다.
- 학생이 직접 체크하는 항목은 숙제 완료 체크와 시험 후 self-check 설문으로 제한한다.

- 나머지 출결, 수업 기록, 숙제 검수, 보강/재시험 판단은 강사가 체크한다.
